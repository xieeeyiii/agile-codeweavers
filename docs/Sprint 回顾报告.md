## McCabe圈复杂度对比

### 计算方法

采用**McCabe圈复杂度公式**：**V(G) = 决策节点数 + 1**

决策节点包括：`if`、`else if`、`switch case`、`for`、`while`、`&&`、`||`

###  重构前复杂度（基于旧代码）

以 `handle_move()` 方法为例：

```python
def handle_move(self, direction):
    # 1. 边界检测
    if (new_x < 0 or new_x >= c.MAP_WIDTH) or (new_y < 0 or new_y >= c.MAP_HEIGHT):
        print("不能走出边界")
        return
    
    # 2. 撞墙检测
    if cell_content == c.WALL:
        print("撞到墙了")
        return
    
    # 3. 拾取物品检测
    if cell_content == c.ITEM:
        self.player.pick_item(cell_content)
        self.maze_map.set_cell(new_x, new_y, c.FLOOR)
    
    # 4. 到达终点检测
    if cell_content == c.GOAL:
        print("游戏胜利")
```

| 决策节点 | 类型 | 数量  |
| :------- | :--- | :---- |
| 边界检测 | if   | 1     |
| 撞墙检测 | if   | 1     |
| 拾取物品 | if   | 1     |
| 到达终点 | if   | 1     |
| **合计** |      | **4** |

**重构前圈复杂度：V = 4 + 1 = 5**

### 重构后复杂度（基于重构方案）

重构后拆分为3个独立类：

**类1：MoveService（移动服务）**
```python
def move(self, new_x, new_y):
    # 1. 边界检测
    if self.is_out_of_bounds(new_x, new_y):
        return False
    # 2. 撞墙检测
    if self.is_wall(new_x, new_y):
        return False
    return True
```
| 决策节点 | 数量  |
| :------- | :---- |
| 边界检测 | 1     |
| 撞墙检测 | 1     |
| **合计** | **2** |

**类2：PickService（拾取服务）**
```python
def pick(self, cell_content):
    if cell_content == c.ITEM:
        return True
    return False
```
| 决策节点 | 数量  |
| :------- | :---- |
| 物品检测 | 1     |
| **合计** | **1** |

**类3：GameService（游戏服务）**
```python
def check_goal(self, cell_content):
    if cell_content == c.GOAL:
        return True
    return False
```
| 决策节点 | 数量  |
| :------- | :---- |
| 终点检测 | 1     |
| **合计** | **1** |

## 重构方案实施

### 运用的设计模式

| 面向对象基石 | 重构前问题                   | 重构后解决方案                            |
| :----------- | :--------------------------- | :---------------------------------------- |
| **封装**     | 移动、拾取、胜利判断混在一起 | 拆分为3个独立服务类，各司其职             |
| **继承**     | 每个命令单独处理，代码重复   | 创建Command抽象父类，4个子类继承          |
| **多态**     | if-else链判断命令类型        | 统一调用`execute()`接口，工厂模式创建命令 |

###  重构前后结构对比

**重构前（上帝类）：**
```
CommandHandler
├── process_input()
├── handle_move()      ← 4个分支
├── show_help()
├── show_inventory()
└── game_running
```

**重构后（职责分离）：**
```
Command (抽象父类)
├── MoveCommand → 依赖 MoveService
├── HelpCommand
├── InventoryCommand
└── QuitCommand

MoveService
├── move()        ← 2个分支
└── is_out_of_bounds()

PickService
└── pick()        ← 1个分支

GameService
└── check_goal()  ← 1个分支
```

### 方法签名规范化

| 类          | 方法     | 签名                                    |
| :---------- | :------- | :-------------------------------------- |
| MoveService | 移动判定 | `move(new_x: int, new_y: int) -> bool`  |
| PickService | 物品拾取 | `pick(cell_content: str) -> bool`       |
| GameService | 胜利判定 | `check_goal(cell_content: str) -> bool` |
| Command     | 命令执行 | `execute() -> None`（抽象方法）         |
